Lesson 4:Network

NETWORK
Network

Network is a group of entities which can communicate each other even though they are not all directly connected.It is a 3000 years old concept.
Different ways to measure Networks: 1. Latency is one way to measure networks. Is basically the time it takes message to get from the source to the destination and is measured in milliseconds(1000 milliseconds = 1 second). 2.Bandwidth: it refer to the amount of information that can be transmitted per unit time, and is measured in bits per second or million bits per second 'Mbps'.

HTTP-Protocol
Protocol

Protocol: Protocol is a set of rules which define how two entities can talk to each other over the Internet in this case. For the web, the protocol HTTP define the client-server communication rules; basically how the client, such as the web browser, is allowed to send and get message to and from the server. HTTP is a very simple protocol which has only two main messages. HTTP Get is one http message; the client can send a message to the server and get an object. The server elaborate the request, extract and send the object back to the client.

Validation

VALIDATION
Validation

Validation is a process to validate data received through HTML form is as expected.The server should not process just any data sent because a user can maliciously corrupt the server.This will prevent other users to use the website. By validating the users input we make sure that our website is in good hands and that way we can make our site secure from the hackers.

Validation for the user experience is also important.If a user enters a wrong information for example maybe a wrong month or day or an year,validation should catch it and ask user for the correct inputs. A good website should keep all the correct fields entered by the user so that user doesn't need to re-enter eveything, thus saving user time,fasturation and avoiding making more errors.Also verifying the user input will make sure the user get to the proper site which will make the user experience better.We can even increase user experience by customizing the webpage for user through validation.

Process :The user make a GET request and the server respond with the form data. After that, user make a POST request to the server with the entered data. In this process the server run the validation functions we define in our web server. At this point the server has 2 possible things: if the data is correct after validation, the server will accept it. However if the data is bad the server will respond with the form data again, including an error message which says the data is not correct.
